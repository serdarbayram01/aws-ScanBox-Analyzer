About the Advice Module

The Advice module consolidates data from three other modules (SecOps, MapInventory, and FinOps) to produce a comprehensive AWS Well-Architected Framework (WAFR) assessment report. It analyzes your AWS account's active services against the six WAFR pillars and provides professional-grade findings, recommendations, and positive observations — just like an AWS Solutions Architect would deliver in an engagement.

The module does NOT make any changes to your AWS account. It only reads existing scan data and Cost Explorer metrics to generate advisory output.

2

Prerequisites

Before running an assessment, you must complete scans in the following modules:

SecOps
Security compliance scan. Provides FAIL/PASS/WARNING findings per service.
MapInventory
Resource inventory scan. Provides detailed resource configuration data.
FinOps
Cost data (fetched live from Cost Explorer). Requires ce:GetCostAndUsage permission.
3

How to Use

  1. Run a SecOps scan for your target profile (SecOps module)
  2. Run a MapInventory scan for the same profile (MapInventory module)
  3. Go to the Advice module, select the profile and one or more regions
  4. Click "Run Assessment" — the module will consolidate all data sources
  5. Review findings per service, export as HTML/CSV/PDF

AWS Well-Architected Framework — 6 Pillars

Security (SEC)
Protect data, systems, and assets. Identity management, encryption, detective controls, and incident response.
Operational Excellence (OPS)
Run and monitor systems. Organization, observability, change management, and continuous improvement.
Reliability (REL)
Recovery from failures, fault isolation, auto-scaling, backup, and disaster recovery planning.
Performance Efficiency (PERF)
Efficient resource selection, monitoring, and optimization as demand changes over time.
Cost Optimization (COST)
Avoid unnecessary costs. Right-sizing, pricing models, budgets, and resource lifecycle management.
Sustainability (SUS)
Minimize environmental impact through efficient hardware, managed services, and optimized resource utilization.
5

Reading the Assessment Report

HIGH Critical issues requiring immediate attention
MEDIUM Important improvements recommended
LOW Best practice suggestions

Each finding includes WAFR control codes (e.g., SEC02, COST03) that link to official AWS documentation. Click any code to view the full best practice guidance.

Each service section contains three parts: Findings (issues detected), Recommendations (actionable advice), and Positive Findings (correctly configured items).

6

Required IAM Permissions

The Advice module reads data from other modules' caches and fetches live cost data. Required permissions:

# Minimum required (for FinOps data):
ce:GetCostAndUsage
budgets:DescribeBudgets
sts:GetCallerIdentity
ec2:DescribeRegions
# Recommended (covers all modules):
arn:aws:iam::aws:policy/ReadOnlyAccess

The SecOps and MapInventory scans require broader read permissions (see their respective guide pages). The Advice module itself only needs Cost Explorer and STS access for live data.

7

SSO Token Renewal

If your AWS profile uses SSO and the token has expired, you'll see a banner with the renewal command:

aws sso login --profile <your-profile-name>
