FinOps — Setup Guide
AWS CLI installation, profile configuration and IAM policy requirements

1 Overview

ScanBox FinOps module reads your existing AWS CLI profiles (~/.aws/credentials and ~/.aws/config) directly. It uses read-only AWS Cost Explorer, EC2, Budgets, and STS API calls to analyze your spending.

All operations are read-only. FinOps never creates, modifies, or deletes any AWS resource. Cost Explorer only works via us-east-1 — the dashboard handles this automatically.

2 AWS CLI Installation

Install the AWS CLI v2 for your operating system. If already installed, skip to step 3.

brew install awscli # Or with the official PKG installer: curl "https://awscli.amazonaws.com/AWSCLIV2.pkg" -o "AWSCLIV2.pkg" sudo installer -pkg AWSCLIV2.pkg -target / # Verify: aws --version
# x86_64 curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip" unzip awscliv2.zip && sudo ./aws/install # ARM64 curl "https://awscli.amazonaws.com/awscli-exe-linux-aarch64.zip" -o "awscliv2.zip" unzip awscliv2.zip && sudo ./aws/install aws --version
# Option 1: winget winget install -e --id Amazon.AWSCLI # Option 2: Download MSI from # https://awscli.amazonaws.com/AWSCLIV2.msi aws --version

3 Configure AWS Profiles

ScanBox automatically discovers all profiles from ~/.aws/credentials and ~/.aws/config. Choose one of the methods below to add profiles.

Step 1: Create an IAM user in AWS Console with programmatic access, then run:

# Default profile $ aws configure AWS Access Key ID [None]: AKIAIOSFODNN7EXAMPLE AWS Secret Access Key [None]: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY Default region name [None]: eu-central-1 Default output format [None]: json

Step 2: Add additional named profiles for different accounts/environments:

# Named profile (e.g. production) $ aws configure --profile production AWS Access Key ID [None]: AKIAI44QH8DHBEXAMPLE AWS Secret Access Key [None]: je7MtGbClwBF/2Tk9nENmXopAXNE/tPEAAEXAMPLE Default region name [None]: eu-west-1 Default output format [None]: json # Named profile (e.g. staging) $ aws configure --profile staging AWS Access Key ID [None]: AKIAIOSFODNN7STAGING1 AWS Secret Access Key [None]: wJalrXUtnFEMI/STAGING/bPxRfiCYEXAMPLEKEY Default region name [None]: us-west-2 Default output format [None]: json

Step 3: Verify the profile works:

$ aws sts get-caller-identity --profile production { "UserId": "AIDACKCEVSQ6C2EXAMPLE", "Account": "123456789012", "Arn": "arn:aws:iam::123456789012:user/finops-reader" }
This creates two files automatically:
~/.aws/credentials — stores access keys
~/.aws/config — stores region and output settings

Step 1: Configure SSO with the interactive wizard:

$ aws configure sso SSO session name (Recommended): my-org SSO start URL [None]: https://my-org.awsapps.com/start SSO region [None]: eu-central-1 SSO registration scopes [sso:account:access]: sso:account:access # Browser will open for authentication... # After login, select the account and role: There are N accounts available to you. Using the account ID 123456789012 There are N roles available to you. Using the role name "ReadOnlyAccess" CLI default client Region [None]: eu-west-1 CLI default output format [None]: json CLI profile name [ReadOnlyAccess-123456789012]: my-org-production

Step 2: Add more SSO profiles for other accounts (repeat the wizard):

$ aws configure sso --profile my-org-staging # Uses existing SSO session, just pick a different account/role

Step 3: Login to SSO before using the profiles:

# Login (opens browser for authentication) $ aws sso login --sso-session my-org # Or login for a specific profile $ aws sso login --profile my-org-production # Verify $ aws sts get-caller-identity --profile my-org-production
This creates the following in ~/.aws/config:
# Generated ~/.aws/config [sso-session my-org] sso_start_url = https://my-org.awsapps.com/start sso_region = eu-central-1 sso_registration_scopes = sso:account:access [profile my-org-production] sso_session = my-org sso_account_id = 123456789012 sso_role_name = ReadOnlyAccess region = eu-west-1 output = json [profile my-org-staging] sso_session = my-org sso_account_id = 987654321098 sso_role_name = ReadOnlyAccess region = eu-central-1 output = json
Important: SSO tokens expire. If ScanBox shows an SSO error banner, run aws sso login --profile <name> to refresh.

You can also edit the config files directly:

# ~/.aws/credentials [default] aws_access_key_id = AKIAIOSFODNN7EXAMPLE aws_secret_access_key = wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY [production] aws_access_key_id = AKIAI44QH8DHBEXAMPLE aws_secret_access_key = je7MtGbClwBF/2Tk9nENmXopAXNE/tPEAAEXAMPLE [staging] aws_access_key_id = AKIAIOSFODNN7STAGING1 aws_secret_access_key = wJalrXUtnFEMI/STAGING/bPxRfiCYEXAMPLEKEY
# ~/.aws/config [default] region = us-east-1 output = json [profile production] region = eu-west-1 output = json [profile staging] region = us-west-2 output = json

Cross-account assume role:

# ~/.aws/config — cross-account role [profile target-account] role_arn = arn:aws:iam::123456789012:role/FinOpsReadOnly source_profile = default region = us-east-1

4 Required Permissions by Service

Cost Explorer (Required)
  • ce:GetCostAndUsage
  • ce:GetCostAndUsageWithResources
  • ce:GetDimensionValues
  • ce:GetCostForecast
  • ce:GetTags
  • ce:ListCostAllocationTags
EC2 (Optional)
  • ec2:DescribeInstances
  • ec2:DescribeRegions
  • ec2:DescribeVolumes
  • ec2:DescribeReservedInstances
Budgets (Optional)
  • budgets:ViewBudget
  • budgets:DescribeBudgets
  • budgets:DescribeBudgetAction
STS (Required)
  • sts:GetCallerIdentity
Organizations (Management Account)
  • organizations:DescribeOrganization
  • organizations:ListAccounts
  • organizations:DescribeAccount
Anomaly Detection (Optional)
  • ce:GetAnomalyMonitors
  • ce:GetAnomalies
  • ce:GetAnomalySubscriptions

5 IAM Policies

FinOpsCore Required
Cost Explorer + STS — sufficient for basic cost dashboard.
ce:GetCostAndUsage ce:GetDimensionValues ce:GetCostForecast sts:GetCallerIdentity
{ "Version": "2012-10-17", "Statement": [ { "Sid": "CostExplorerReadOnly", "Effect": "Allow", "Action": [ "ce:GetCostAndUsage", "ce:GetCostAndUsageWithResources", "ce:GetDimensionValues", "ce:GetCostForecast", "ce:GetTags", "ce:ListCostAllocationTags" ], "Resource": "*" }, { "Sid": "STSCallerIdentity", "Effect": "Allow", "Action": ["sts:GetCallerIdentity"], "Resource": "*" } ] }
FinOpsFull Recommended
All features: EC2 inventory, budgets, anomaly detection. Still read-only.
ce:* ec2:Describe* budgets:ViewBudget sts:GetCallerIdentity
{ "Version": "2012-10-17", "Statement": [ { "Sid": "CostExplorerFullRead", "Effect": "Allow", "Action": [ "ce:GetCostAndUsage", "ce:GetCostAndUsageWithResources", "ce:GetDimensionValues", "ce:GetCostForecast", "ce:GetTags", "ce:GetAnomalyMonitors", "ce:GetAnomalies", "ce:GetAnomalySubscriptions", "ce:ListCostAllocationTags", "ce:ListCostCategoryDefinitions" ], "Resource": "*" }, { "Sid": "EC2ReadOnly", "Effect": "Allow", "Action": [ "ec2:DescribeInstances", "ec2:DescribeRegions", "ec2:DescribeVolumes", "ec2:DescribeReservedInstances" ], "Resource": "*" }, { "Sid": "BudgetsReadOnly", "Effect": "Allow", "Action": [ "budgets:ViewBudget", "budgets:DescribeBudgets", "budgets:DescribeBudgetAction" ], "Resource": "arn:aws:budgets::*:budget/*" }, { "Sid": "STSCallerIdentity", "Effect": "Allow", "Action": ["sts:GetCallerIdentity"], "Resource": "*" } ] }
FinOpsOrganizations Management Account
Extra policy for AWS Organizations management accounts.
{ "Version": "2012-10-17", "Statement": [ { "Sid": "OrganizationsReadOnly", "Effect": "Allow", "Action": [ "organizations:DescribeOrganization", "organizations:ListAccounts", "organizations:DescribeAccount" ], "Resource": "*" } ] }
Quick attach (AWS CLI): Save the policy to a file, then attach it:
# Inline policy — user aws iam put-user-policy \ --user-name finops-reader \ --policy-name FinOpsFull \ --policy-document file://FinOpsFull.json # Managed policy — role aws iam create-policy \ --policy-name FinOpsFull \ --policy-document file://FinOpsFull.json aws iam attach-role-policy \ --role-name FinOpsRole \ --policy-arn arn:aws:iam::ACCOUNT_ID:policy/FinOpsFull

6 Verification

Test that your profile is configured correctly:

# 1. Verify identity aws sts get-caller-identity --profile your-profile-name # 2. Test Cost Explorer access aws ce get-cost-and-usage \ --profile your-profile-name \ --region us-east-1 \ --time-period Start=2026-01-01,End=2026-02-01 \ --granularity MONTHLY \ --metrics "BlendedCost"
If both commands return results, your profile is ready. Click Dashboard in the left menu to start analyzing costs.
