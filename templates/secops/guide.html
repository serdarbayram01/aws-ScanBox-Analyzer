SecOps — Setup Guide
IAM permissions, profile configuration and compliance frameworks for security scanning

1 Overview

SecOps scans your AWS account using read-only boto3 API calls across 24 services — no agents, no third-party tools, no write permissions required. It produces compliance scores against CIS, HIPAA, ISO 27001, and WAFR frameworks.

All checks are read-only. SecOps never creates, modifies, or deletes any AWS resource.

2 AWS CLI Installation

AWS CLI v2 must be installed before configuring profiles. If already installed, skip to step 3.

# Option 1: Homebrew (recommended) $ brew install awscli # Option 2: Official PKG installer $ curl "https://awscli.amazonaws.com/AWSCLIV2.pkg" -o "AWSCLIV2.pkg" $ sudo installer -pkg AWSCLIV2.pkg -target / # Verify $ aws --version aws-cli/2.x.x Python/3.x.x Darwin/...
# x86_64 $ curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip" $ unzip awscliv2.zip && sudo ./aws/install # ARM64 (Graviton) $ curl "https://awscli.amazonaws.com/awscli-exe-linux-aarch64.zip" -o "awscliv2.zip" $ unzip awscliv2.zip && sudo ./aws/install # Verify $ aws --version
# Option 1: winget PS> winget install -e --id Amazon.AWSCLI # Option 2: Download MSI installer from # https://awscli.amazonaws.com/AWSCLIV2.msi # Verify PS> aws --version

5 Configure AWS Profile

Set up an AWS CLI profile for SecOps scanning. Choose one of the methods below.

Step 1: Run the interactive configuration wizard:

$ aws configure --profile secops-scanner AWS Access Key ID [None]: AKIAIOSFODNN7EXAMPLE AWS Secret Access Key [None]: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY Default region name [None]: us-east-1 Default output format [None]: json

Step 2: Verify the profile works:

$ aws sts get-caller-identity --profile secops-scanner { "UserId": "AIDACKCEVSQ6C2EXAMPLE", "Account": "123456789012", "Arn": "arn:aws:iam::123456789012:user/secops-reader" }

Step 1: Configure SSO with the interactive wizard:

$ aws configure sso SSO session name (Recommended): my-org SSO start URL [None]: https://my-org.awsapps.com/start SSO region [None]: eu-central-1 SSO registration scopes [sso:account:access]: sso:account:access # Browser opens for authentication... # After login, select account and role: Using the account ID 123456789012 Using the role name "SecOpsReadOnly" CLI default client Region [None]: us-east-1 CLI default output format [None]: json CLI profile name [SecOpsReadOnly-123456789012]: secops-scanner

Step 2: Login before scanning:

$ aws sso login --sso-session my-org # Verify $ aws sts get-caller-identity --profile secops-scanner
Important: SSO tokens expire. If ScanBox shows an SSO error banner, run aws sso login --profile secops-scanner to refresh.

For scanning a target account from a source account using sts:AssumeRole:

Step 1: In the target account, create a role with the SecOpsReadOnly policy and trust the source account.

Step 2: In the source account, add sts:AssumeRole permission.

Step 3: Configure the profile:

# ~/.aws/config [profile target-account-secops] role_arn = arn:aws:iam::123456789012:role/SecOpsReadOnly source_profile = my-admin-profile region = us-east-1

Step 4: Verify:

$ aws sts get-caller-identity --profile target-account-secops { "UserId": "AROA3XFRBF23:botocore-session", "Account": "123456789012", "Arn": "arn:aws:sts::123456789012:assumed-role/SecOpsReadOnly/botocore-session" }

6 Supported Compliance Frameworks

SecOps maps every finding to one or more compliance frameworks. Each scan produces scores per framework.

WAFR — Well-Architected
  • Security
  • Operational Excellence
  • Reliability
  • Performance Efficiency
  • Cost Optimization
  • Sustainability
CIS AWS Foundations v3
  • Identity & Access Management
  • Storage
  • Logging
  • Monitoring
  • Networking
HIPAA Security Rule
  • 164.308 — Administrative Safeguards
  • 164.312 — Technical Safeguards
ISO/IEC 27001:2022
  • A.9 — Access Control
  • A.10 — Cryptography
  • A.12 — Operations Security
  • A.13 — Communications Security

7 Verification

Test that your profile has all required permissions for SecOps scanning:

# 1. Verify identity $ aws sts get-caller-identity --profile secops-scanner # 2. Test IAM access $ aws iam get-account-summary --profile secops-scanner # 3. Test S3 access $ aws s3api get-public-access-block --profile secops-scanner # 4. Test CloudTrail access $ aws cloudtrail describe-trails --profile secops-scanner
If all commands return results without AccessDenied errors, your profile is ready. Click Dashboard in the left menu to start scanning.
