Map Inventory — Setup Guide
Features, required IAM permissions, and configuration for resource inventory scanning

1 Overview

Map Inventory scans your AWS account using read-only boto3 API calls to discover and catalog all resources across 150+ AWS services. It produces a comprehensive inventory map showing what resources exist, where they are, and how they are configured.

All operations are read-only. Map Inventory never creates, modifies, or deletes any AWS resource.

2 Key Features

150+ AWS Services

Scans all major AWS services including EC2, S3, RDS, Lambda, ECS, EKS, DynamoDB, CloudFront, and 140+ more.

Multi-Profile Support

Scan multiple AWS profiles simultaneously. Compare up to 2 profiles side-by-side with matching/unique service breakdown.

Interactive Dashboard

Radar, bar, donut charts. Filter by service, region, type. Click any service to drill down into resource details.

Export Reports

Generate HTML, CSV, and PDF reports grouped by service. Share inventory snapshots with your team.

Default Resource Filtering

Automatically identifies AWS default resources (default VPC, default security groups, etc.) and excludes them from results.

Persistent Cache

Scan results are cached per profile. View previous scans instantly without re-scanning. Clear cache when needed.

5 Configure AWS Profile

Add a named profile in ~/.aws/credentials or use aws configure --profile:

# Option A — ~/.aws/credentials [my-inventory-profile] aws_access_key_id = AKIAIOSFODNN7EXAMPLE aws_secret_access_key = wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY region = us-east-1 # Option B — AWS CLI aws configure --profile my-inventory-profile

6 SSO & Cross-Account Roles

For SSO profiles, ensure the permission set includes read-only access. For cross-account scanning via sts:AssumeRole, add sts:AssumeRole to the source profile and attach the MapInventoryReadOnly policy to the target role.

# ~/.aws/config — SSO profile [profile my-sso-account] sso_start_url = https://my-org.awsapps.com/start sso_region = us-east-1 sso_account_id = 123456789012 sso_role_name = ReadOnlyAccess region = us-east-1 # Assume-role profile [profile target-account-inventory] role_arn = arn:aws:iam::123456789012:role/MapInventoryReadOnly source_profile = my-admin-profile region = us-east-1
If your SSO token expires, the app will detect this and show the exact aws sso login --profile <name> command to run.

7 How It Works

  • Profile Selection: Choose one or more AWS profiles. Star profiles as favorites for quick access.
  • Background Scan: The scan runs in a background thread using 40 parallel workers. Progress is shown in real-time.
  • Global vs Regional: Services like IAM, S3, Route53 are scanned once (global). All others are scanned per enabled region.
  • Default Resources: AWS creates default resources (VPC, subnets, security groups) in every region. These are flagged and excluded by default — toggle the checkbox to include them.
  • Caching: Results are cached per profile. The cache persists until you explicitly clear it. No auto-expiry.
  • Comparison: Select exactly 2 profiles and scan. The comparison view shows matching services on top and unique services below.
  • Error Handling: If a service can't be accessed (missing permissions, service not available in region), it's skipped and marked as failed in the scan statistics.
